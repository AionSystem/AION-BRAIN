## Purpose

This directory defines the **constitutional foundation** of the AION-BRAIN project.

It establishes the **non-negotiable definitions, laws, boundaries, and invariants** that constrain all architectures, engines, experiments, and future extensions within this repository.

Everything below this layer must comply with what is defined here.
Nothing in this layer is overridden by implementation convenience, performance goals, or emergent capabilities.

If a system behavior conflicts with this layer, the behavior is invalid by definition.

## What This Layer Is

This is **not** documentation.
This is **not** implementation guidance.
This is **not** a policy suggestion.

This is a **foundational constraint layer** that:

- Defines what autonomy means (and does not mean)
- Establishes explicit red lines for AI capability and authority
- Encodes authored laws governing system behavior
- Anchors definitions in time with authorship and versioning
- Prevents silent capability drift through interpretation

This layer exists so that future systems cannot claim ambiguity.

## Design Principles

1. **Definitions precede behavior**  
   No system behavior is evaluated without a prior definition.

2. **Authority is not emergent**  
   Capability does not imply permission.

3. **Ambiguity is a failure mode**  
   If something can be interpreted two ways, it will be exploited.

4. **Boundaries are explicit or they do not exist**  
   Silence is not neutrality.

5. **Authorship is recorded**  
   All laws and definitions include timestamps and versioning.

---

## Change Control

This layer is **append-only by design**.

- Existing definitions are not deleted
- Laws are superseded, not erased
- Amendments require explicit authorship and timestamps
- Historical context is preserved

Future capability does not retroactively justify reinterpretation.
