## Guidelines

### What to Share

| Good | Better | Best |
|------|--------|------|
| "I used Oracle Layer" | "I used Oracle Layer for X" | "I used Oracle Layer for X and achieved Y results" |
| Brief description | Detailed implementation | Full case study with lessons |
| Screenshot | Demo link | Open source code |

### Expectations

- **Honesty** — Be truthful about results and limitations
- **Attribution** — Credit any collaborators or sources
- **Relevance** — Projects should use AION-BRAIN meaningfully
- **Helpfulness** — Share insights that help others

### What Not to Share

- Projects that misuse AION-BRAIN
- Confidential client work (without permission)
- Incomplete work without explanation
- Promotional content without substance

---

## Feedback Guidelines

### Giving Feedback

| Good Feedback | Poor Feedback |
|---------------|---------------|
| "Have you considered X for better Y?" | "This is wrong" |
| "In my experience, this approach helped with..." | "I don't like it" |
| "Nice work! One suggestion..." | Just criticism |
| Specific, actionable | Vague, general |

### Receiving Feedback

- **Be open** — Feedback helps you improve
- **Ask questions** — Clarify if you don't understand
- **Say thanks** — Acknowledge helpful feedback
- **Iterate** — Use feedback to improve
