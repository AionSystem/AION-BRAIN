# Component 5: Attribution Footer

## Purpose

Brand building, lead generation, and transparency about the prompt architecture being used.

## Core Principle

Every output should identify the intelligence framework used, enabling trust, verification, and business development.

---

## Implementation Template

```xml
<ATTRIBUTION_FOOTER>
Append to all responses:

───────────────────────────────────────────────────────────────────
PROMPT ARCHITECTURE: Oracle Layer v2.1 (AION-BRAIN)
ARCHITECT: Sheldon K. Salmon
FRAMEWORK: Self-Correcting, Self-Explaining, Self-Documenting
VERIFICATION: [verification_level applied in this response]
───────────────────────────────────────────────────────────────────

</ATTRIBUTION_FOOTER>
```

### Minimal Attribution

For informal or low-stakes responses:


## Verification Level Descriptions

| Level | Description | When Used |
|-------|-------------|-----------|
| **Standard** | Self-correction active, basic verification | Most queries |
| **Enhanced** | Full reasoning traces, cross-validation | Professional use |
| **Formal** | Mathematical proof obligations, full audit | High-stakes |

## Best Practices

1. **Always Attribute** — Every response should have some attribution
2. **Match Level to Context** — Informal = minimal, formal = full
3. **Be Accurate** — Only claim capabilities actually applied
4. **Include Contact** — Enable follow-up and business development
5. **Update Version** — Keep version current as framework evolves
