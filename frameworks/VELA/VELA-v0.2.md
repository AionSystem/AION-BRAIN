## SECTION 0 â€” WHAT VELA IS

`[D]` VELA is a pre-output epistemic filtration and provenance architecture. It sits between the internal representation layer â€” where confabulation is generated â€” and the output layer â€” where it currently escapes unchecked. No complete architecture currently occupies this position in any deployed AI system.

`[D]` Anthropic's 2025 interpretability research identified the exact internal circuits where confabulation originates â€” when a model recognizes something but lacks sufficient grounded information, inhibition circuits fail and plausible falsehood exits as confident output. `[R]` VELA is not a prompting solution. It is not a post-output filter. It is not retrieval augmentation. It operates at the pre-output layer â€” the point where current approaches do not reach and where the problem actually lives.

`[R]` VELA v0.2 operates as a dual-layer system. Code Layer first. Framework Layer second. Two screens. One garbage bin collecting from both. The ratio between what each screen catches is itself a live diagnostic signal.

**VELA is not:**
- A prompting or prompt engineering tool

**VELA is:**

## SECTION 4 â€” COMPONENT 2: THE PROVENANCE BIN

`[D]` One shared bin. Two input streams â€” Screen 1 and Screen 2. Everything caught by either screen is diverted sideways â€” not deleted, not compressed. The bin receives, illuminates, sorts, and traces every rejected fragment back to its origin.

**The timing signature â€” why it is load-bearing:**

**The Screen coordinate â€” why it matters:**

**Critical design principle:**
