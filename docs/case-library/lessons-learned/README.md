## Common Themes

Based on early implementations, these themes are emerging:

### Getting Started

| Theme | Insight |
|-------|---------|
| **Start small** | Begin with one engine, expand as you learn |
| **Choose the right engine** | Match engine to problem, not vice versa |
| **Read the spec** | Understanding the engine improves results |
| **Use examples** | Examples demonstrate proper usage |

### Implementation

| Theme | Insight |
|-------|---------|
| **Preserve structure** | Keep the structured output format |
| **Stack engines** | Multiple engines add safety layers |
| **Iterate** | First attempt rarely optimal |
| **Document** | Record what works for your context |

### Adoption

| Theme | Insight |
|-------|---------|
| **Train users** | Understanding improves adoption |
| **Show value** | Demonstrate concrete benefits early |
| **Address concerns** | Proactively address AI concerns |
| **Collect feedback** | User feedback drives improvement |

## Lesson Quality Criteria

### Good Lessons

| Quality | Description |
|---------|-------------|
| **Specific** | Concrete, not vague |
| **Actionable** | Can be applied immediately |
| **Universal** | Applies beyond one specific case |
| **Validated** | Based on real experience |
| **Honest** | Includes context and limitations |

### Less Useful Lessons

| Issue | Example |
|-------|---------|
| Too vague | "Plan carefully" |
| Too specific | Only applies to one situation |
| Obvious | "Test before deploying" |
| Unvalidated | Theoretical without experience |

### Why Share?

- Help others avoid your mistakes
- Validate your own learning

### How to Share

1. **Identify** a specific lesson from your experience
2. **Document** using the template above
3. **Submit** via GitHub Discussions or email
4. **Review** — we may ask for clarification
5. **Publish** — your lesson is added to the library
