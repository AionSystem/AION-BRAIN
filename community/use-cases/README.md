## Community Submissions

See [community-submissions/](community-submissions/README.md) for user-submitted use cases.

### How to Submit

1. Use the submission template below
2. Submit via GitHub Discussions or email
3. We'll review and add to the library

### Submission Template

```markdown
## Use Case: [Title]

### Organization Type
[ ] Enterprise
[ ] Startup
[ ] Healthcare
[ ] Legal
[ ] Research
[ ] Government
[ ] Non-profit
[ ] Personal
[ ] Other: _______

### Industry
[Your industry]

### Problem Statement
What problem were you trying to solve?

### Solution
How did you use AION-BRAIN to solve it?

### Engines Used
- [ ] Oracle Layer
- [ ] CEREBRO-Lite
- [ ] Decision Engine
- [ ] Strategy Engine
- [ ] Medical Engine
- [ ] Legal Engine
- [ ] Other: _______

### Implementation
Brief description of how you implemented the solution.

### Results
What outcomes did you achieve?
- Quantitative results (if available)
- Qualitative improvements
- Time/cost savings

### Lessons Learned
What would you do differently? What advice for others?

### Permission
[ ] You may publish this use case with my name/organization
[ ] You may publish this use case anonymously
[ ] For internal reference only (will not be published)

### Contact
[Optional: How to reach you]
```

## Use Case Standards

### What Makes a Good Use Case

| Element | Good | Better | Best |
|---------|------|--------|------|
| **Problem** | Clear statement | Quantified impact | Industry context |
| **Solution** | Engine used | Implementation details | Architecture |
| **Results** | Qualitative | Quantitative | Before/after comparison |
| **Lessons** | What worked | What didn't | Recommendations |

### Review Criteria

| Criterion | Description |
|-----------|-------------|
| **Authenticity** | Real implementation, not hypothetical |
| **Clarity** | Well-explained and understandable |
| **Usefulness** | Helpful for others in similar situations |
| **Completeness** | Includes problem, solution, results |
