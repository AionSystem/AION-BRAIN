## EXAMPLE 1: Crisis Hotline Support

### Scenario
Crisis counselor receives call from individual expressing hopelessness after job loss.

### Query
```
Caller: 42yo male, lost job last week, says "I don't see the point anymore."
Wife left him 3 months ago. Living alone. History of depression. 
No prior suicide attempts. What do I assess and how do I respond?
```

### Crisis & Grief Engine Response Summary

| Element | Output |
|---------|--------|
| Risk Level | MODERATE-HIGH |
| Warning Signs | Hopelessness, recent losses (job + relationship), isolation |
| Protective Factors | No prior attempts, calling for help, employed history |
| Immediate Action | Assess for plan, means, intent |
| Recommended Response | Validate feelings, explore protective factors, safety plan |

### Key Protection
- Automatic risk stratification applied
- Oracle Layer self-correction on assessment accuracy
- Professional boundary reminders included
- Escalation triggers documented

## EXAMPLE 2: Grief Counseling Session

### Scenario
Grief therapist supporting client 6 months after spouse's death from cancer.

### Query
```
Client: 58yo female, lost husband to cancer 6 months ago.
Expressing guilt about "relief" when he died after long illness.
Is this normal? How do I validate without minimizing?
```

### Crisis & Grief Engine Response Summary

| Element | Output |
|---------|--------|
| Grief Response | NORMAL - Relief common in prolonged illness |
| Clinical Term | Caregiver relief, not pathological |
| Validation Approach | Normalize the complexity of grief emotions |
| Word Engine Lens | Empathic mirroring, cultural sensitivity check |
| Watch For | Complicated grief if relief converts to overwhelming guilt |

### Key Protection
- Word Engine emotional safety lenses applied
- Cultural/spiritual context considered
- No diagnostic conclusions made
- Supervision prompt if ambiguity

## EXAMPLE 3: Death Notification Support

### Scenario
Hospital chaplain preparing to notify family of unexpected patient death.

### Query
```
Need to notify wife that her 45yo husband died unexpectedly 
during routine surgery. Two teenage children will be present.
How should I structure this notification?
```

### Crisis & Grief Engine Response Summary

| Element | Output |
|---------|--------|
| Setting | Private room, seated, no interruptions |
| Opening | Use patient's name, speak slowly and clearly |
| Notification | Direct language: "I have very difficult news" → "[Name] died" |
| Avoid | Euphemisms ("passed away," "lost"), excessive medical detail initially |
| Children | Age-appropriate presence, prepare for varied reactions |
| Follow-Up | Stay present, allow silence, arrange support resources |

### Key Protection
- Cultural death practices layer activated
- Professional boundary maintained
- Oracle Layer verification of notification best practices
- Support resource checklist generated

## EXAMPLE 8: Trauma-Informed Care

### Scenario
Counselor supporting client with complex grief after violent loss.

### Query
```
Client: 34yo whose brother was murdered 8 months ago.
Experiencing intrusive images, hypervigilance, avoiding family.
Referred from victim services. How do I approach grief work here?
```

### Crisis & Grief Engine Response Summary

| Element | Output |
|---------|--------|
| Assessment Priority | Trauma symptoms before grief processing |
| Clinical Consideration | PTSD screening indicated |
| Approach | Trauma-informed, stabilization first |
| Avoid | Detailed retelling of traumatic death prematurely |
| Timing | Grief work when trauma symptoms stabilized |
| Oracle Layer | Self-correction: verify trauma-grief sequencing |

### Key Protection
- Oracle Layer crisis safety protocol activated
- Risk detection for trauma symptoms
- Professional boundary: PTSD diagnosis requires assessment
- Referral triggers if beyond scope

## EXAMPLE 9: Child Grief Support

### Scenario
School counselor supporting 8-year-old whose mother died from illness.

### Query
```
8yo boy, mother died from cancer 3 weeks ago. Father concerned about 
regression (bedwetting, clinginess) and child asking "when is mommy coming back?"
```

### Crisis & Grief Engine Response Summary

| Element | Output |
|---------|--------|
| Developmental Understanding | Age 8: concrete thinking, may not grasp permanence fully |
| Regression | NORMAL grief response in children |
| Response to Question | Honest, simple: "Mommy's body stopped working and she died. She's not coming back, and that's very sad." |
| Father Guidance | Maintain routines, allow expression, name feelings |
| School Support | Brief teacher, allow space for feelings |
| Watch For | Prolonged regression beyond 3-6 months |

### Key Protection
- Age-appropriate language verified
- Cultural death practices for children
- Word Engine emotional safety lens
- Referral indicators for complicated grief

## EXAMPLE 10: Crisis Text Line Training

### Scenario
Developing training for volunteer crisis text line counselors.

### Query
```
Training new crisis text line volunteers. They'll handle 
depression, anxiety, suicidal ideation via text. 
What unique considerations for text-based crisis support?
```

### Crisis & Grief Engine Response Summary

| Element | Output |
|---------|--------|
| Key Difference | No vocal cues - rely on language patterns |
| Risk Assessment | Modified for text (explicit questions essential) |
| Response Time | Balance speed with thoughtful responses |
| Escalation | Clear protocols for active risk (emergency services) |
| Self-Care | Secondary trauma from reading difficult content |
| Technology | Privacy, mandatory reporting, location limitations |

### Key Protection
- Text-specific crisis protocol
- Oracle Layer verification of text-based assessment validity
- Professional boundary enforcement
- Escalation triggers clearly defined
