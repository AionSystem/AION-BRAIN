## Why Apache 2.0?

### Strategic Selection Rationale

| Factor | Apache 2.0 Advantage |
|--------|---------------------|
| **Commercial Adoption** | Permissive terms enable enterprise deployment |
| **Patent Protection** | Express patent grant reduces legal uncertainty |
| **Modification Freedom** | No copyleft requirements for derivative works |
| **Attribution Balance** | Reasonable attribution without excessive burden |
| **Industry Standard** | Widely accepted in enterprise and open-source communities |
| **Legal Clarity** | Well-understood terms reduce compliance complexity |

## What Apache 2.0 Requires

### Mandatory Conditions

#### 1. License Inclusion

You must include a copy of the Apache 2.0 license in any distribution:

```
Location: LICENSE file in root directory
Content: Full text of Apache License 2.0
Format: Plain text preferred
```

#### 2. Copyright Notice Preservation

Retain all copyright notices from original files:

```
Example:
Copyright 2025 AION-BRAIN
Licensed under the Apache License, Version 2.0
```

#### 3. NOTICE File Inclusion

If a NOTICE file exists, you must:

- Include the NOTICE file in distributions
- Preserve all attributions in the NOTICE file
- Add your own notices if appropriate (clearly marked)

#### 4. State Changes

When modifying files, you must:

```
Add a statement indicating modifications:
"Modified by [Your Name/Organization], [Date]"
```

#### 5. Trademark Restrictions

The license does NOT grant trademark rights:

- No right to use "AION-BRAIN" trademark
- No right to use "AION" trademark
- No right to use associated logos
- Separate trademark license required for trademark use

## What Apache 2.0 Does NOT Require

### No Copyleft Obligations

Unlike GPL licenses, Apache 2.0 does NOT require:

| Not Required | Explanation |
|--------------|-------------|
| **Source Code Disclosure** | No obligation to share your modifications |
| **Same License for Derivatives** | Derivative works can use different licenses |
| **Contribution Back** | No requirement to upstream changes |
| **Public Availability** | No obligation to make work publicly available |

### No Warranty Obligations

Apache 2.0 provides software "AS IS" without:

- Warranty of merchantability
- Warranty of fitness for particular purpose
- Warranty of non-infringement
- Any other warranty, express or implied

## Common Questions

### Q: Can I use AION-BRAIN in commercial products?

**A: Yes.** Apache 2.0 explicitly permits commercial use. You must comply with license conditions (attribution, license inclusion, etc.).

### Q: Do I have to share my modifications?

**A: No.** Apache 2.0 is permissive, not copyleft. You are not required to share modifications or make source code available.

### Q: Can I charge for products using AION-BRAIN?

**A: Yes.** You may charge for products incorporating AION-BRAIN materials. Apache 2.0 places no restrictions on pricing.

### Q: Can I use the AION-BRAIN name in my product?

**A: Requires permission.** The license does not grant trademark rights. Contact AIONSYSTEM@outlook.com for trademark licensing.

### Q: What happens if I modify files?

**A: Add modification statement.** Include a notice that you have changed the file, e.g., "Modified by [Name], [Date]."

### Q: Can I combine Apache 2.0 code with other licenses?

**A: Usually yes.** Apache 2.0 is compatible with most permissive and proprietary licenses. Check compatibility for GPL combinations.

### Q: What about patents?

**A: Patent grant included.** Apache 2.0 includes an express patent license covering contributor patents in the work. This terminates if you sue over patents.
