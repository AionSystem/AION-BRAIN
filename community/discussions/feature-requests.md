Propose new features, improvements, and enhancements for AION-BRAIN.

## How to Submit a Feature Request

### Before Submitting

1. **Search existing requests** — Your idea may already be proposed
2. **Check the roadmap** — It may already be planned: [Roadmap](../../docs/roadmap.md)
3. **Consider scope** — Is this a core feature or a derivative engine?

### Submission Format

Use this template for clear, actionable requests:

```markdown
## Feature Request: [Descriptive Title]

### Category
[ ] New Engine
[ ] Engine Enhancement
[ ] Documentation
[ ] Infrastructure
[ ] Community
[ ] Other: ________

### Problem Statement
What problem does this feature solve? Who experiences this problem?

### Proposed Solution
How would you like this solved? Be as specific as possible.

### Alternatives Considered
What other approaches have you considered? Why is this approach better?

### Impact Assessment
- Who benefits from this feature?
- How significant is the improvement?
- What's the effort level? (Low/Medium/High)

### Additional Context
Screenshots, examples, references, or any other relevant information.
```

## Voting System

### How Voting Works

- **Comment with support** — Add "+1" and your use case
- **Comment with concerns** — Share potential issues
- **Priority influences** — High-vote items get priority consideration

### Vote Guidelines

| Good Vote | Bad Vote |
|-----------|----------|
| "+1 We need this for our hospital deployment" | "+1" (no context) |
| "Important for compliance, here's why..." | "Me too" |
| "Alternative approach: ..." | Unrelated comment |

## Prioritization Framework

### How We Prioritize

| Factor | Weight | Description |
|--------|--------|-------------|
| **Safety Impact** | 30% | Does it improve AI safety? |
| **User Demand** | 25% | How many users want this? |
| **Strategic Fit** | 20% | Does it align with vision? |
| **Feasibility** | 15% | Can we build it well? |
| **Resources** | 10% | Do we have capacity? |

### What Gets Prioritized

- Features that improve safety
- High-demand requests with clear use cases
- Strategic features aligned with roadmap
- Feasible within current resources

### What Gets Deprioritized

- Features misaligned with core principles
- Low-demand or niche requests
- Technically infeasible requests
- Features better suited for derivatives
